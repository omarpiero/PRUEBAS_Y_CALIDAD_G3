# Evaluación Técnica y Estructural: ISO/IEC 29119 (Software Testing)

> **Norma:** ISO/IEC 29119 - Estándar Internacional para Pruebas de Software.
> **Proyecto:** JM y JS Alimentos LMS v2.0
> **Directorio de Referencia:** `tests/Feature/`, `tests/Unit/`, `phpunit.xml`
> **Fecha de Evaluación:** Junio 2026

La conformidad del LMS v2.0 con la norma ISO/IEC 29119 es comprobable y transparente. Se refleja directamente en la suite de pruebas automatizadas y en los reportes generados desde el framework (ej. archivos en `screenshots_p/`).

## 1. Procesos de Pruebas: Organización y Aislamiento (ISO/IEC 29119-2)
- **Entorno Limpio y Seguro:** El archivo `phpunit.xml` sobreescribe el driver de base de datos forzando el uso de una base de datos temporal cargada en RAM (`sqlite: memory`). Se inyecta el trait `RefreshDatabase` en las clases de test para instanciar el esquema de bases de datos desde cero por cada método ejecutado.
- **Resultados Probados:** Las pruebas se han orquestado exitosamente cubriendo **89 métodos funcionales** con **426 aserciones atómicas**, con un 100% de éxito de integración.

## 2. Clasificación de Diseño de Pruebas (ISO/IEC 29119-4)

### A. Pruebas de Caja Negra (Validaciones de Extremo a Extremo)
Tests agrupados para interactuar netamente mediante HTTP y DOM requests. Representaron 34 test funcionales y 158 aserciones:
- `PaymentStripeTest.php`: Verifica que un carrito vacío reacciona correctamente redirigiendo al catálogo. Comprueba la firma HMAC.
- `PublicCourseCatalogTest.php`: Evalúa que solo los cursos en estado `published` se proyecten en el DOM, evaluando las compuertas hacia la capa de presentación.
- `StudentCourseAccessTest.php`: Verifica las barreras de autenticación entre visitantes invitados y estudiantes matriculados.

### B. Pruebas de Caja Blanca (Pruebas Estructurales y Middleware)
Agrupadas en 21 test exitosos con 109 aserciones, evalúan las barreras de acceso:
- `AdminSecurityAndRolesTest.php`: Comprueba el límite de intentos (Throttle) y la imposibilidad de que el último administrador asuma un rol inferior (prevención de *lockout*).
- `PermissionMiddlewareTest.php`: Valida a bajo nivel que usuarios no autorizados no puedan instanciar rutas protegidas.

### C. Pruebas Unitarias y de Servicios Integrados
Evalúan métodos aislados (12 pruebas, 43 aserciones) o integraciones complejas externas (22 pruebas restantes, 116 aserciones):
- **Stubs/Mocks de Pasarela:** En transacciones financieras, se interceptan las conexiones reales (Mocking) usando falsificaciones a los eventos para nunca tocar la red.
- **Mocking de IA (`GeminiAssistantTest`):** Uso de `Http::fake()` para testear el chatbot sin consumir la API oficial.
- **VideoEmbedServiceTest:** Pruebas atómicas aisladas a las conversiones Regex de URLs de Vimeo y YouTube.

## 3. Cobertura Temática (Negative Testing)
A nivel global, la suite no solo comprueba casos "felices". Un fuerte énfasis recae en las pruebas negativas para disparar Excepciones (Códigos 422, 403 y 500 gestionados) certificando la resiliencia (Ej. pagos con carrito vacío o cupones expirados).
